よく使うMermaidパターン
1. フローチャート
```mermaid flowchart TD A[事実を記録] --> B{重要？} B -->|はい| C[Issue化] B -->|いいえ| D[Daily Noteに残す] C --> E[抽象化] E --> F[#permanent に昇格] ```
2. マインドマップ
```mermaid mindmap root((知的生産)) 読書 Literature Notes 抽象化 転用 実装 Daily Note Issue化 Permanent化 執筆 記事構成 Issue参照 公開 ```
3. タイムライン
```mermaid timeline title 知識の成長プロセス 2025-01-20 : Daily Note : 一時メモ 2025-01-23 : Issue #102 : Literature Note 2025-01-27 : Issue #102 更新 : 抽象化追加 2025-02-03 : Issue #150 : Permanent Note ```

Issue #100: パフォーマンス最適化の原則 ## 核心 動的な必要がないデータは、静的生成すべき。 リアルタイム性と速度のトレードオフを常に意識する。 ## 図解 ```mermaid graph TD A[データの性質] --> B{リアルタイム性必要？} B -->|はい| C[SSR: 遅いが最新] B -->|いいえ| D[SSG: 速い] C --> E[ISR: バランス] ``` ## 原理 - SSR: 毎回サーバーで生成（遅いが最新） - SSG: ビルド時に生成（速いが古い可能性） - ISR: 定期的に再生成（速度と鮮度のバランス） ## 転用 - Webアプリ全般のキャッシュ戦略 - モバイルアプリのデータ同期 ## 元ネタ #95, #87, #72
