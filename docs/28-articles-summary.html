🔄 実践ワークフロー

📝 Zettelkasten 3ステップ

  1. Step 1: Fleeting Notes（一時メモ）
    • 日付フォルダに作成: 2025-01-23.md
    • 思いついたことをとにかく書く（整理不要）
  2. Step 2: Literature Notes（文献メモ）
    • 情報源を明記: 出典: [[書籍名]]
    • 引用と自分の解釈を分ける
  3. Step 3: Permanent Notes（永久メモ）
    • 1ノート1コンセプト原則
    • ファイル名: 概念そのもの（例: 抽象化思考.md）
    • 最低3つの関連ノートとリンク

🎯 事実→抽象化→転用フレームワーク

テンプレート:
## 事実 [具体的な出来事・経験を記録] ## 抽象化 - ここから学べること: - 一般化できる原理: - 本質的なパターン: ## 転用 - 応用できる場面1: - 応用できる場面2: - 応用できる場面3: ## リンク [[関連ノート1]] [[関連ノート2]]

⏱️ 3ヶ月習得ロードマップ

1ヶ月目: 基礎
  • 毎日10分、Daily Noteを書く
  • [[リンク]]を5回以上使う
  • Graph Viewを毎日1回見る
2ヶ月目: 構造化
  • フォルダ構成を決定（例: Inbox/Permanent/Archive）
  • タグ体系を3-5個に絞る
  • 週1回、過去のメモを見返して永久メモ化
3ヶ月目: 自動化
  • Templaterでテンプレート作成
  • Dataviewでクエリ作成
  • 自分専用のワークフロー確立

🔌 必須プラグイン7選と使い方

1. Templater - テンプレート自動化
設定手順:
  1. Settings → Community Plugins → Templater をインストール
  2. Template folder: Templates/ に設定
  3. Hotkey設定: Cmd+T
使用例:
--- created: <% tp.date.now("YYYY-MM-DD") %> tags: #fleeting --- # <% tp.file.title %> ## メモ ## リンク
2. Dataview - 動的クエリ
よく使うクエリ:
```dataview TABLE file.ctime as "作成日" FROM #permanent SORT file.ctime DESC LIMIT 10 ``` ```dataview LIST FROM [[現在のノート]] WHERE contains(file.outlinks, this.file.link) ```
3. Excalidraw - 手書き図解
ショートカット:
  • V - 選択ツール
  • R - 矩形
  • D - ダイヤモンド
  • A - 矢印
  • T - テキスト
  • Cmd+K - Obsidianノートへリンク
効率化のコツ:
  • ライブラリを作成: よく使う図形を保存
  • カラーパレット: 3色に固定（メイン・アクセント・背景）
4. ExcaliBrain - マインドマップ自動生成
使い方:
  1. 任意のノートを開く
  2. コマンドパレット: ExcaliBrain: Start
  3. リンク関係が自動的に図解される
5. Suggester - 爆速ノート作成
設定例:
// Templaterと組み合わせ <% await tp.user.suggester( ["Daily Note", "Permanent Note", "Literature Note"], ["daily", "permanent", "literature"] ) %>
  • ノート作成時にテンプレート選択を自動化
  • タグ入力を候補から選択
6. Calendar - 日次メモ管理
  • 右サイドバーにカレンダー表示
  • 日付クリックで Daily Note 自動作成
  • ドットで書き込み状況を可視化
7. Tag Wrangler - タグ管理
  • タグ一括リネーム
  • タグ階層の可視化
  • 未使用タグの削除

📊 図解・可視化テクニック

🎨 Visual Zettelkasten 実践

手順:
  1. 図解ファイル作成
    • ファイル名: 永久メモ名.excalidraw.md
    • 永久メモと同じフォルダに配置
  2. リンクで紐付け
    • 永久メモ内: ![[永久メモ名.excalidraw.md]]
    • 図解内: Cmd+K で永久メモへリンク
  3. 図解パターン
    • フローチャート: プロセスを図示
    • 概念マップ: 関係性を図示
    • タイムライン: 時系列を図示

🗺️ Graph View 活用法

フィルタ設定:
# 永久メモだけ表示 tag:#permanent # 特定トピックのみ path:Projects/ OR tag:#project # 孤立ノードを除外 -file:""
色分け:
  • Settings → Graph View → Groups
  • tag:#permanent → 紫
  • tag:#fleeting → グレー
  • tag:#literature → 青

📈 情報の構造化テクニック

MOC (Map of Content) 作成:
  1. MOC - トピック名.md ファイル作成
  2. 関連ノートをリスト化:
    # プログラミング MOC ## 基礎 - [[変数とデータ型]] - [[制御構造]] - [[関数]] ## 設計 - [[オブジェクト指向]] - [[デザインパターン]] ## 実践 - [[リファクタリング]] - [[テスト駆動開発]]
  3. 各ノートにMOCへの逆リンク追加
